
# Домашнее задание к занятию «Микросервисы: масштабирование»

Вы работаете в крупной компании, которая строит систему на основе микросервисной архитектуры.
Вам как DevOps-специалисту необходимо выдвинуть предложение по организации инфраструктуры для разработки и эксплуатации.

## Задача 1: Кластеризация

Предложите решение для обеспечения развёртывания, запуска и управления приложениями.
Решение может состоять из одного или нескольких программных продуктов и должно описывать способы и принципы их взаимодействия.

Решение должно соответствовать следующим требованиям:
- поддержка контейнеров;
- обеспечивать обнаружение сервисов и маршрутизацию запросов;
- обеспечивать возможность горизонтального масштабирования;
- обеспечивать возможность автоматического масштабирования;
- обеспечивать явное разделение ресурсов, доступных извне и внутри системы;
- обеспечивать возможность конфигурировать приложения с помощью переменных среды, в том числе с возможностью безопасного хранения чувствительных данных таких как пароли, ключи доступа, ключи шифрования и т. п.

Обоснуйте свой выбор.

В качестве решения для развёртывания и эксплуатации микросервисов предлагаю использовать **Kubernetes**.

### Используемые компоненты

- Оркестрация контейнеров: **Kubernetes**
- Контейнеризация: **Docker**
- Входной трафик и маршрутизация: **NGINX Ingress Controller**
- Обнаружение сервисов: **Kubernetes Service (DNS)**
- Автоматическое масштабирование: **Horizontal Pod Autoscaling (HPA)**
- Хранение конфигурации: **ConfigMap**
- Хранение секретов: **Kubernetes Secrets**
- Мониторинг метрик: **Prometheus**

---

## Описание взаимодействия

Приложения упаковываются в Docker-контейнеры и разворачиваются в Kubernetes в виде `Deployment`.  
Каждый сервис публикуется через объект `Service`, обеспечивающий service discovery и балансировку нагрузки.  
Внешний доступ к системе осуществляется через NGINX Ingress Controller.  
Масштабирование Pod'ов выполняется автоматически с помощью HPA на основе метрик Prometheus.  
Конфигурация и чувствительные данные передаются приложениям через переменные среды.

---

## Соответствие требованиям

- **Поддержка контейнеров**  
  Kubernetes управляет Docker-контейнерами и их жизненным циклом.

- **Обнаружение сервисов и маршрутизация**  
  Service Discovery реализован через Kubernetes Service и встроенный DNS, маршрутизация внешнего трафика — через Ingress.

- **Горизонтальное масштабирование**  
  Масштабирование выполняется за счёт увеличения количества реплик Pod'ов.

- **Автоматическое масштабирование**  
  HPA автоматически изменяет число реплик на основе загрузки.

- **Разделение внешних и внутренних ресурсов**  
  Внешний трафик поступает только через Ingress, внутренние сервисы доступны исключительно внутри кластера.

- **Конфигурация и безопасное хранение данных**  
  ConfigMap используется для параметров конфигурации, Secrets — для паролей, токенов и ключей.

---

## Обоснование выбора

Kubernetes является промышленным стандартом для микросервисной архитектуры. Он обеспечивает отказоустойчивость, автоматическое масштабирование и единый подход к управлению инфраструктурой на всех средах.
